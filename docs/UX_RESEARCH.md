## 2. 用户痛点（基于 GitHub Issues / Reddit r/AndroidApps / Play 评论）

1. **找不到账号**：账号一多就要肉眼翻列表 → 需要**搜索框** + **置顶/收藏**
2. **复制验证码不便**：长按点按要去外部输入框粘贴，且**剪贴板不会自动清** → **点按复制 + 倒计时清剪贴板**
3. **30 秒切换瞬间复制到旧码**：粘贴时已经过期 → **剩余 ≤ 5s 时复制下一码**或**预显示下一码**
4. **解锁太频繁**：放下手机 3 秒切回还要重新解锁/指纹 → **可配置宽限期 0/15/30/60s**
5. **误删账号无救**：滑动删除一去不回 → **撤销 Snackbar / 软删除 7 天**
6. **看一眼就被人看到**：通勤场景显示 6 位码很尴尬 → **隐藏码，点按显形 N 秒**
7. **拖拽排序难用**：当前只能编辑 sortOrder 字段 → **长按拖拽**
8. **大图标识别慢**：纯文字"Google / GitHub" 一眼难辨 → **本地内置常见 issuer 字母色块图标**（不联网）

## 4. 安全/红线复核（Sam 预审）

| 子任务 | 红线影响 | 评审结论 |
|---|---|---|
| T1 搜索 | 仅前端过滤，不涉及加密路径 | ✅ 通过 |
| T2 剪贴板 | 写入剪贴板 = 短暂明文，30s 后回收；Android 13+ 系统通知里显示，需要打 `EXTRA_IS_SENSITIVE` | ⚠️ 必须 set sensitive flag |
| T3 隐藏验证码 | 减少肩窥风险，**改善**安全姿态 | ✅ 通过 |
| T4 下一码预览 | 不会让"曾经有效的码"被未授权读取 | ✅ 通过 |
| T5 宽限期 | **降低**安全强度 → 必须默认 = 0（即刻锁定），仅由用户主动放宽 | ⚠️ 默认值必须为 0 |
| T6 撤销删除 | 软删除窗口期内 secret 仍在 DB 中，未提升攻击面 | ✅ 通过 |
| T7 字母图标 | 纯本地绘制，不联网 | ✅ 通过 |

## 5. 验收用例（Tina 预先列）

- TC-T1-01：搜索 "goo" 应过滤出所有包含 google / GoOgleX 的条目（大小写不敏感）
- TC-T2-01：点按验证码后剪贴板含验证码字符串，30 秒后自动清空（且只清自家写入的内容）
- TC-T2-02：Android 13+ 通知中不应回显验证码内容（sensitive flag）
- TC-T3-01：隐藏开关开启时，列表项默认显示 `••••••`，点按切换为真实码 5s 后再次隐藏
- TC-T4-01：剩余秒数 ≤5 时 next code 文本可见且与下一周期生成的码一致
- TC-T5-01：宽限期=30s 时，进入后台 20s 返回不需要解锁；进入后台 31s 返回需要解锁
- TC-T5-02：默认值=0，未升级偏好的老用户行为与原版一致
- TC-T6-01：滑动删除后弹 Snackbar，点撤销后账号恢复且 secret 一致
- TC-T7-01：issuer 为空时图标显示账号首字母；非 ASCII 字符不崩溃
- TC-T8-01：长按账号上下拖拽改变顺序，松手后顺序持久化；杀进程重开仍保留
- TC-T8-02：搜索框非空时长按不应触发拖拽（避免基于过滤态写回乱序）
- TC-S1-01：进入「设置」页修改隐藏码 / 下一码 / 宽限期；返回主列表立即生效
- TC-S1-02：设置页继承 BaseSecureActivity，release 包内不能截屏；从后台回前台触发解锁路由
- TC-T9-01：长按账号选择“置顶常用”后，条目上提到列表顶部且 issuer 前出现 ★
- TC-T9-02：跨分区（收藏<->非收藏）拖拽被拒绝，顺序不变
- TC-T9-03：DB 从 v1/v2 升级到 v3，所有老账号 favorite 默认 false，原顺序保留
- TC-T10-01：设备语言为 English 时，所有页面文案仅及列表、设置、备份、锁页全部是英文
- TC-T10-02：上下文字符串 fmt_remaining_seconds、fmt_next_code 在英文下参数位置与表达正确
